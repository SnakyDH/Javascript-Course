Características

  1. Lenguaje de Alto Nivel

  2. Interpretado

    3. El interprete es el navegador.

  3. Dinámico

  4. Débil Tipado

  5. Multi paradigma

  6. Sensible a MAYUSCULAS y minusculas

    7. Puede ser una buena practica o no, depende de tu lugar de trabajo.

  7. No necesitas los puntos y comas al final de cada linea

Escritura de código

  1. Los identifacdores deben comenzar con:

    • Una Letra o Signo de dolar o un guion bajo / Nunca numeros ni caracteres especiales

  2. Usar snake_case = Archivos

  3. Usar UPPER_CASE = Constantes

  4. Usar UpperCamelCase = Clases !== Metodos || Atributos

  5. Usar lowerCamelCase = Objetos || Primitivos (Variables) || Funciones || Instancias

  6. A: abstract
    B: boolean, break, byte
    C: case, catch, char, class, const, continue
    D: debugger, default, delete, do, double
    E: else, enum, export, extends
    F: false, final, finally, float, for, function
    G: goto
    I: if, implements, import, in, instanceof, int, interface
    L: let, long
    N: native, new, null
    P: package, private, protected, public
    R: return
    S: short, static, super, switch, synchronized
    T: this, throw, throws, transient, true, try, typeof
    V: var, volatile, void
    W: while, with

Ordenamiento de código

  1. IMPORTACIÓN DE MÓDULOS

  2. DECLARACIÓN DE VARIABLES

  3. DECLARACIÓN DE FUNCIONES

  4. EJECUCIÓN de CÓDIGO
